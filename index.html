Here is the loop again
- The Loop takes care of what posts to actually get, and works differently on each template
- Now let's looks into the anatomy of it

have_posts()

If: If there are posts

While: Loop through each post


if ( have_posts() )  {
	while ( have_posts() ) { ... }
}
This function lets us loop through every post that is returned
- On our page template, that's just the current page
- Later we'll see how this can return many posts

the_post()

Sets up the current post object

It is the first declaration in The Loop


while ( have_posts() ) {
	the_post();
	...
This function will let us use aspects of the post like its title, permalink, featured image, etc
- Without it WordPress will think we're dealing with different content
- Make sure it's the first thing you define because you don't want to forget it

Anatomy of WP_Query()


$the_query = new WP_Query( array( /* query parameters */ ) );
// The Loop
if ( $the_query->have_posts() ) {
	echo '
    ';
	while ( $the_query->have_posts() ) {
		$the_query->the_post();
		echo '
  • ' . get_the_title() . '
    • ';
	}
	echo '
';
} else {
// no posts found
}
// Restore original Post Data
wp_reset_postdata();
This is a straight forward WP_Query.
- It's small, but like Spiderman, has great responsibility

The query object

The result of the query is set to an object variable ($the_query)


$the_query = new WP_Query( array( /* query parameters */ ) );


while ( $the_query->have_posts() ) { ... }


$the_query->the_post();
the_title();
Basically, the WP_Query function is an easy way to write complex database queries
- It takes a ton of parameters to figure out what posts you want to work with and sets those to a PHP obj
* There are a lot of useful functions within the object like have_posts which lets you loop through the results of the query
* and the_post(), which, you'll remember, sets up the current post

Resetting the Post Data

This function must be run after the query loop


$the_query = new WP_Query( $args );
if ( $the_query->have_posts() ) {
	while ( $the_query->have_posts() ) { ... }
}
wp_reset_postdata(); /* UBER IMPORTANT !!! */
This is really important
- If not run, WordPress will assume that we are still using data from the last post returned by the query

Hooks: Actions and Filters

codex.wordpress.org says:

Hooks are provided by WordPress to allow your plugin to 'hook into' the rest of WordPress...There are 2 kinds of hooks:
  • Actions: A custom PHP function defined in your plugin (or theme) and hooked, i.e. set to respond, to specific events that take place in WordPress.
  • Filters: Functions that WordPress passes data through, at certain points in execution, just before taking some action with the data

Adding functionality

wp_enqueue_style / wp_enqueue_script

Load WordPress' core copy of jQuery. If you're using jQuery with your theme you should always use WP's copy vs. an external version. 

 
 
function my_scripts() {
	wp_enqueue_script( 'jquery');
}

add_action( 'wp_enqueue_scripts', 'my_scripts' );
 

Adding functionality

wp_enqueue_style / wp_enqueue_script

Load a script that depends on jQuery. Bootstrap is a framework that needs jQuery to run. The code below will load both jQuery and Bootstrap JS.


 
function add_bootstrap() {
	wp_enqueue_script(
		'add-bootstrap-script',
		get_stylesheet_directory_uri() . 
		'/inc/bootstrap/js/bootstrap.min.js',
		array( 'jquery' )
	);
}

add_action( 'wp_enqueue_scripts', 'add_bootstrap' );
 

Adding functionality

wp_enqueue_style / wp_enqueue_script

Load a stylesheet and a script.
